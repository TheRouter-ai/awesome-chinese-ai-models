# This Week in China AI Models (2026-W27)

**Period:** June 23 – June 29, 2026

**Overview:**
Week 27 was unusually broad: Tencent Hunyuan pushed image generation forward with **HunyuanImage-3.0**, DeepSeek published **DeepSeek-V4 DSpark** Flash and Pro checkpoints on Hugging Face, and Qwen expanded in two directions at once with **Qwen3-ASR** for multilingual speech recognition and **Qwen-AgentWorld** for agent/world-model research. On the infrastructure side, InternLM shipped **lmdeploy v0.14.0** with FP8 KV-cache quantization and Qwen3 Omni support, while DeepSeek opened **DeepSpec**, a full-stack codebase for speculative decoding research. ## New Models & Releases

**DeepSeek-V4 DSpark** — DeepSeek published two Hugging Face checkpoints under the DeepSeek-V4 line:
- **[DeepSeek-V4-Flash-DSpark](https://huggingface.co/deepseek-ai/DeepSeek-V4-Flash-DSpark)** — an 8-bit, endpoint-compatible text-generation checkpoint tagged with `deepseek_v4` and `safetensors`.
- **[DeepSeek-V4-Pro-DSpark](https://huggingface.co/deepseek-ai/DeepSeek-V4-Pro-DSpark)** — the companion Pro variant, also MIT-licensed and tied to arxiv:2606.19348.

The pair looks positioned as a deployable Flash/Pro split: one model optimized for faster serving and another for stronger capability, both released in a format that should be straightforward for downstream inference stacks to pick up.

**Qwen3-ASR** — Qwen released a new open-source speech recognition family through both GitHub and Hugging Face. The [Qwen3-ASR repository](https://github.com/QwenLM/Qwen3-ASR) describes multilingual speech, music, and song recognition with language detection and timestamp prediction. Initial Hugging Face checkpoints include:
- **[Qwen3-ASR-0.6B-hf](https://huggingface.co/Qwen/Qwen3-ASR-0.6B-hf)**
- **[Qwen3-ASR-1.7B-hf](https://huggingface.co/Qwen/Qwen3-ASR-1.7B-hf)**
- **[Qwen3-ForcedAligner-0.6B-hf](https://huggingface.co/Qwen/Qwen3-ForcedAligner-0.6B-hf)**

Language tags include Chinese, English, Cantonese, Arabic, and French depending on the checkpoint, making this one of Qwen's more practical releases for media transcription and alignment workflows.

**Qwen-AgentWorld-35B-A3B** — Qwen published **[Qwen-AgentWorld-35B-A3B](https://huggingface.co/Qwen/Qwen-AgentWorld-35B-A3B)** alongside the [Qwen-AgentWorld](https://github.com/QwenLM/Qwen-AgentWorld) repository. The project frames the model as a language world model for general agents: rather than only responding to prompts, it targets environment simulation and agent training/evaluation loops. The checkpoint uses a Qwen3.5 MoE-style tag and is categorized for image-text-to-text use.

**Tencent HunyuanImage-3.0** — Tencent Hunyuan updated **[HunyuanImage-3.0](https://github.com/Tencent-Hunyuan/HunyuanImage-3.0)**, described as a powerful native multimodal model for image generation. The release keeps Tencent active on the image-generation frontier shortly after its HY-Embodied robotics push in the previous week.

## Tooling & Infrastructure

**DeepSeek DeepSpec** — DeepSeek opened **[DeepSpec](https://github.com/deepseek-ai/DeepSpec)**, a full-stack codebase for training and evaluating speculative decoding algorithms. Speculative decoding is becoming a core serving optimization for large models, so a first-party DeepSeek implementation is notable for teams trying to reproduce or extend fast-decoding pipelines.

**InternLM/lmdeploy v0.14.0** — The stable **[lmdeploy v0.14.0](https://github.com/InternLM/lmdeploy/releases/tag/v0.14.0)** release landed after the previous alpha. Highlights include FP8 KV-cache quantization for lower-memory long-context serving and support for Qwen3 Omni. This is a practical infrastructure update for production inference teams tracking Chinese model stacks.

**Qwen Code v0.19.x** — Qwen Code continued its high-frequency release cycle with stable releases through **[v0.19.2](https://github.com/QwenLM/qwen-code/releases/tag/v0.19.2)** plus preview/nightly builds. Notable changes across the v0.19 line include remote LSP serving work, MCP resource completion/discovery improvements, and web-fetch JSON fallback fixes. The project also published **[cua-driver-rs v0.6.8](https://github.com/QwenLM/qwen-code/releases/tag/cua-driver-rs-v0.6.8)** prebuilt binaries for the vendored computer-use driver.

**OpenBMB PilotDeck Desktop 260623** — OpenBMB released **[PilotDeck Desktop 260623](https://github.com/OpenBMB/PilotDeck/releases/tag/v260623)**, focused on stability, IM channels, frontend experience, and always-on/cron-style workflow improvements.

## Research & Evaluation

**Qwen Confident-Decoding** — Qwen updated **[Confident-Decoding](https://github.com/QwenLM/Confident-Decoding)**, adding another decoding-related research thread to the week alongside DeepSeek's DeepSpec.

**InternLM EndoCoT** — InternLM published **[EndoCoT](https://github.com/InternLM/EndoCoT)**, the official implementation for an ECCV 2026 paper on scaling endogenous chain-of-thought reasoning in diffusion models for complex structured generation.

**OpenBMB SHIFT** — OpenBMB updated **[SHIFT](https://github.com/OpenBMB/SHIFT)**, code for “SHIFT: Gate-Modulated Activation Steering for Knowledge Conflict Mitigation in Retrieval-Augmented Generation.” The project targets a practical RAG pain point: what to do when retrieved evidence conflicts with a model's parametric knowledge.
