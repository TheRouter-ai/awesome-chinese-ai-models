## New Models & Releases

**Tencent HiLS-Attention-7B** — Tencent published **[HiLS-Attention-7B](https://huggingface.co/tencent/HiLS-Attention-7B)** alongside the **[Tencent-Hunyuan/HiLS-Attention](https://github.com/Tencent-Hunyuan/HiLS-Attention)** repository. The project describes HiLS-Attention as a hierarchical sparse-attention method for efficient long-context modeling, using learned chunk selection rather than paying the full attention cost. The model card tags the checkpoint for long-context, sparse attention, efficient attention, text generation, and pretraining, making this the week's most important foundation-model-adjacent release.

**Tencent R3 embedding and rerank models** — Tencent added **[R3-embedding-0.6b](https://huggingface.co/tencent/R3-embedding-0.6b)** and **[R3-rerank-0.6b](https://huggingface.co/tencent/R3-rerank-0.6b)**. Both are small Qwen3-derived retrieval models tagged for `agent-skill-retrieval`; the embedding model is based on Qwen3-Embedding-0.6B, while the reranker is based on Qwen3-Reranker-0.6B. The pairing suggests a practical retrieval stack for agent memory, tool selection, or skill/library search rather than a headline chat model.

## Long-Context & Retrieval Research

**HiLS-Attention codebase** — The **[HiLS-Attention](https://github.com/Tencent-Hunyuan/HiLS-Attention)** repository is positioned as official code for “Hierarchical Sparse Attention Done Right: Toward Infinite Context Modeling.” The README frames the method as chunk-wise sparse attention trained end-to-end under the language-modeling loss. For teams watching long-context serving costs, this is worth tracking because it targets both long-context extrapolation and faster inference while trying to preserve short- and medium-context quality.

**Agent-skill retrieval becomes explicit** — The R3 models are notable because their metadata names `agent-skill-retrieval` directly. That is a small but useful signal: Chinese model teams are increasingly publishing not only chat checkpoints, but also retrieval components intended to make agent systems more reliable and modular.

## Tooling & Ecosystem

**Qwen Code v0.19.7 → v0.19.9** — Qwen Code kept its fast release cadence with **[v0.19.7](https://github.com/QwenLM/qwen-code/releases/tag/v0.19.7)**, **[v0.19.8](https://github.com/QwenLM/qwen-code/releases/tag/v0.19.8)**, and **[v0.19.9](https://github.com/QwenLM/qwen-code/releases/tag/v0.19.9)**, plus several nightly and preview builds. The updates were mostly product-hardening work: stronger PR triage gates, WeCom/channel documentation, session/history-chain handling, command matching, configurable memory-agent timeouts, browser automation gating, vision bridge retries, web-shell polish, and fixes for repeated subagent tool-call loops.

**Qwen Code Action** — Qwen opened **[qwen-code-action](https://github.com/QwenLM/qwen-code-action)**, described as a GitHub Action that integrates Qwen Code into development workflows. Together with the v0.19.x releases, this points to Qwen Code moving from a standalone coding assistant toward CI and repository automation.

**cua-driver-rs v0.7.1** — Qwen Code also published **[cua-driver-rs v0.7.1](https://github.com/QwenLM/qwen-code/releases/tag/cua-driver-rs-v0.7.1)**, a prebuilt relative-coordinate fork vendored under `packages/cua-driver`, with macOS binaries described as signed and notarized. It is a lower-level release, but relevant for teams following computer-use automation inside coding-agent workflows.
